# PR Log - 2026-04-08

## 작업 내용: 선택(s) 모드 중복 선택 방지 및 UI 정밀 조정

### 1. 기능 개선

- **선택 우선순위 조정**: 텍스트와 도형이 겹쳐 있는 경우, 도형(화살표, 사각형 등)을 텍스트보다 우선하여 선택하도록 로직을 변경했습니다.
- **선택 로직 통합**: 기존에 `MouseDown`(도형)과 `Click`(텍스트)으로 분리되어 있던 선택 로직을 `startDraw`(MouseDown) 하나로 통합하여 중복 선택 현상을 근본적으로 해결했습니다.
- **텍스트 선택 정밀도 강화**: 텍스트 박스의 인식 영역(Padding)을 기존 24px에서 **4px**로 대폭 축소했습니다. 이제 글자 주변의 빈 공간이 아닌 실제 텍스트가 적힌 부분만을 클릭해야 편집기가 열립니다.
- **텍스트 박스 UI 및 드래그 경험 최적화**: 
  - **공백 완전 제거**: `line-height`(1.1)와 패딩(1px)을 극한으로 줄여 박스 하단의 불필요한 보라색 여백 문제를 해결했습니다.
  - **디자인 컴팩트화**: 하단 투명도 조절 UI의 너비를 220px에서 **175px**로 줄이고, 슬라이더에 고정 너비(**w-[80px]**)를 적용하여 컨테이너 이탈 문제를 수정했습니다.
  - **실시간 드래그 동기화**: 텍스트 박스를 드래그할 때 캔버스상의 실제 텍스트가 실시간으로 따라다니도록 구현하여 이동 위치를 즉각 확인 가능하게 했습니다.

### 2. 기술적 세부 사항

- `PdfViewer.tsx`의 `startDraw` 함수에서 도형 히트 테스트 실패 시에만 텍스트 히트 테스트를 수행하도록 순차적 순서를 강제했습니다.
- `handleMouseMove`에서 `isDraggingBox` 상태일 때 `textAnnotations` 배열을 즉시 업데이트하여 렌더링 동기화를 구현했습니다.
- 에디터 최소 높이를 24px에서 **18px**로 추가 하향 조정했습니다.
- 슬라이더 UI에 `w-[80px]` 고정 너비를 할당하여 flex 레이아웃 내에서의 크기 불일치 문제를 해결했습니다.
